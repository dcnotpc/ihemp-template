## Architecture Decision: Dynamic State Configuration
**User clarified**: Migrating from 18 WordPress sites to 18 Vercel sites (no WordPress remains).

**Solution implemented**: 
- Created dynamic state configuration system in `src/config/state.ts`
- Reads `NEXT_PUBLIC_STATE` environment variable (default: "colorado")
- Dynamically loads corresponding state data from `src/data/states/{stateSlug}.ts`
- Generates state-specific: domain, logo path, tagline, descriptions, URLs
- State-specific overrides supported (e.g., Colorado has special resource URLs)

**Deployment model**: Separate Vercel project per state, using same codebase with environment variable switching.
